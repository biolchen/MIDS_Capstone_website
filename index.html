    OUR GOAL

Problem Statement

A conventional machine learning, often referred to as Predictive learning”, requires a vast number of labelled images. However, such supervised pre-training approach is limited because hiring humans to label data is expensive. It would be great if our algorithm is able to leverage the vast number of unlabeled images on the internet and somehow obtain a meaningful representation on its own. It would be great if an algorithm can learn the vast number of unlabeled images and somehow obtain a meaningful model for future reference. This is exactly contrastive learning is trying to solve. The most popular algorithm for constrastive learning is cycleGAN. However, a new model, named C.U.T., was reported to be faster and more efficient. We would like to compare these two algorithms.

Our Goal

Our main goal is to Test CUT to see if it outperforms cycleGan at domain adaptation in satellites images. We will test the effectiveness of Contrastive Learning for Unpaired Image-to-Image Translation (C.U.T.) when used for domain adaptation in satellite imagery. Our approach is to apply domain adaptation, converting images taken in the summer into synthetic spring, fall, and winter images. We will compare the results of a model trained with synthetic images generated with C.U.T with the results of a model trained with images generated with CycleGAN.

Our experimental plan

The pipeline

Step1:Baseline Validation Accuracy

We begin with a labeled training set consisting of summer images and a labeled validation set consisting of winter images. We train a model using the summer images and then try to predict winter images with this model. This results in a baseline evaluation accuracy.

Results

Step2:Generate Fake Winter Images

We then train the CUT model with the labeled summer images and randomly selected winter images. This results in synthetic versions of our summer images that have been “winterized”, but retain the original labels. This creates a batch of fake labeled winter images.

Examples

Step3:Retrain Model and Compare Results

We then retrain our classification model with the original labeled summer images combined with the new fake labeled winter images. The model is then used to predict the validation winter images, resulting in a new validation score. If there is an improvement in validation accuracy, it is a result of the domain adaptation through the CUT model.

The BigEarthNet dataset

The BigEarthNet is a new large-scale Sentinel-2 benchmark archive, consisting of 590,326 Sentinel-2 image patches. To construct the BigEarthNet, 125 Sentinel-2 tiles acquired between June 2017 and May 2018 over the 10 countries (Austria, Belgium, Finland, Ireland, Kosovo, Lithuania, Luxembourg, Portugal, Serbia, Switzerland) of Europe were initially selected. All the tiles were atmospherically corrected by the Sentinel-2 Level 2A product generation and formatting tool (sen2cor). Then, they were divided into 590,326 non-overlapping image patches. Each image patch was annotated by the multiple land-cover classes (i.e., multi-labels) that were provided from the CORINE Land Cover database of the year 2018 (CLC 2018).

Reference:

G. Sumbul, M. Charfuelan, B. Demir, V. Markl, BigEarthNet: A large-scale benchmark archive for remote sensing image understanding, arXiv preprint, 2019.
